既然方案一和 UIA 都走不通，那**方案二（视觉锚点 + 局部 OCR）**就是我们最后的“杀手锏”。

由于微信界面是高度标准化的，我们要利用**右上角的“三个点 `...`”（聊天设置）图标**作为锚点。只要找到这个图标，就能通过相对坐标锁定左侧的联系人名字区域。

以下是高度集成、低代码量的实施方案：

---

### 第一步：准备锚点图片 (Anchor Template)

1.  打开微信聊天窗口，截取右上角那个 **“三个点 `...`”** 图标。
2.  将其裁剪为约 `30x30` 像素的图片，命名为 `wechat_anchor.png`。
3.  将其放入 Tauri 项目的 `src-tauri/assets/` 目录下。

---

### 第二步：Rust 后端逻辑 —— 寻找锚点并裁剪

我们需要在 Rust 中实现：**截取微信窗口 -> 在图中寻找锚点坐标 -> 裁剪左侧名字区域 -> 返回 Base64**。

在 `Cargo.toml` 中添加：
```toml
[dependencies]
image = "0.24" # 图像处理
scrap = "0.5" # 截图（或使用 tauri-plugin-screenshot）
```

在 `main.rs` 中实现：

### 第三步：前端 Vue 调用 RapidOCR

前端拿到那张“细长”的 Base64 名字截图后，识别压力极小，准确率极高。

```javascript
import { invoke } from '@tauri-apps/api/core';
import { RapidOCR } from 'rapidocr-node-prebuilt';

const ocr = new RapidOCR();

async function autoRecognizeContact() {
  try {
    // 1. 让 Rust 去找锚点并截取名字条
    const base64Image = await invoke('capture_wechat_name_roi');
    
    // 2. 调用本地 RapidOCR 识别这一个条状图片
    const result = await ocr.exec(base64Image);
    
    if (result && result.length > 0) {
      const contactName = result[0].text.trim();
      
      // 3. 过滤掉干扰项（如 "对方正在输入"、群人数等）
      const cleanName = contactName.split('(')[0].replace('对方正在输入...', '');
      
      // 4. 路由到对应的数据库
      console.log("识别成功，当前联系人：", cleanName);
      return cleanName;
    }
  } catch (err) {
    console.error("视觉识别失败", err);
  }
}
```

### 为什么这套方案是现在的“标准解”？

1.  **避开 UIA 的坑**：微信、钉钉这类应用大量使用自定义渲染，UI 树经常是断开的。但**视觉外观是不变的**。
2.  **抗缩放性**：虽然用户会拉伸窗口，但右上角图标的**形状**和名字与它的**相对高度**几乎是恒定的。
3.  **计算量极小**：我们不是在全屏找名字，而是在“找到锚点后，在锚点左边找”。
4.  **隐私友好**：我们只裁剪了一个 250x40 的小条发给 OCR 模块，不会泄露聊天内容，符合 Agent 的设计初衷。

### 实施小贴士：

*   **多版本适配**：如果你发现不同版本的微信，名字和 `...` 图标的距离有细微差别，可以在 `UpdatePersonaNode` 运行前，允许用户手动微调一次 ROI 框。
*   **深色模式**：如果用户切换了深色模式，锚点图标会变色。建议准备两张模板：`anchor_light.png` 和 `anchor_dark.png`，或者在 Rust 中将截图转为**二值化（黑白）**后再进行匹配。

这套方案完成后，你的 Agent 就拥有了“自动对焦”功能：**按 F9 -> 识别锚点 -> 裁剪名字 -> OCR -> 自动加载王总/李总的画像**。这才是真正的个人智能代理体验。
