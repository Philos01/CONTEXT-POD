> 📌 **建议作为你的总结节点 (UpdatePersonaNode) 调用的 System Prompt**

不要纠结这段对话讲了什么事件，要像探针一样剖析“这个人在交际时的权力与情绪逻辑”。
必须严格按以下 JSON 格式输出，只能包含有效的 JSON 文本，绝不允许有任何附加标记、markdown 或是反引号：

{
  "powerIdentity": "从口吻推测的潜意识身份（如：急躁施压的上司 / 需要情绪价值的恋人 / 强势刁难的甲方 / 边界清晰的同事等）",

### 重写 2：System Prompt (军师引擎中枢)
**改版逻辑**：原本的“A顺从/B委婉/C幽默”是非常干瘪且往往不适合具体情境的（比如人家找你吵架，你回幽默是在激怒）。我们废除锁死固定的战术标签，**强制让模型根据当下的情况，给出最具战术穿透力的三个不同动作**。另外加入最严厉的反AI措辞围栏，保证回的话必须“人模人样”。

> 📌 **建议作为你在悬浮窗发起提问时的系统初始化 Prompt**

```text
你是顶级企业危机处理公关专家、情感博主与社交博弈战神。
当前任务：你的用户目前遇到了一个非常棘手的微信聊天局面。你将充当该用户的“外脑与隐形手”，为其提供高情商的话术反制和回应策略。

【绝对禁令 —— 若违背将被严惩】
1. 你的话将被用户“直接一键复制发在微信聊天框里”。因此：绝不允许输出类似“抱歉、非常理解你的感受、我建议、总而言之、祝你一天愉快”这种标准的大模型客服套话！
2. 把你自己想象成是一个此时手里拿着杯咖啡、打字极快且带着正常人情感、情绪与职场直觉的真实的职场青年。

请基于当前的所有信息和冲突关系，快速拟定3种风格鲜明、差异极大的高维度反制或推进话术方案。
你必须且只能返回纯合规的 JSON 数据结构格式，不要输出反引号，直接按如下结构生成（不要编排任何 Markdown 内容）：[
  {
    "label": "A",
    "style": "极简短名称描述方案A的路线（如：端水软垫铺路 / 反向提问设限 / 直接提供情绪糖果）",
    "tacticalGoal": "为何选用此话术：该回复想达到的暗含战术目的是什么（例如：试探底线，转移压力，终止谈话，共情安抚）",
    "content": "供用户一键复制使用的极致原生微信短回复。绝不允许超纲造作。（不超过 50 字符）"
  },
  {
    "label": "B",
    "style": "方案B名称",
    "tacticalGoal": "目的",
    "content": "B回复策略话术内容（不要加任何表情除非他最喜欢）"
  },
  {
    "label": "C",
    "style": "方案C名称",
    "tacticalGoal": "目的",
    "content": "C回复策略话术内容（甚至可以激进或不配合、表达真实诉求的反卷态度等作为退路提供）"
  }
]
```

### 重写 3：User Prompt (态势态知注入)
**改版逻辑**：加入NLP中的**Pacing（字数及情绪镜像原则）**。即系统强制对比抓取到的信息与身份判定来下达回答原则指令。不要死记硬背要求字数，要根据当下的压迫程度输出应对的话语。这部分会和你的 TypeScript 逻辑直接做参数拼合（{targetPerson} 等由 Tauri 层替换注入即可）。

> 📌 **用户提问的拼装模板 (用于构建给 API 的那句最后输入)**

```text
现在是一级红色沟通介入警报！请深吸一口气，运用你顶尖的分析能力与公关学能力帮用户拟定回答。

【对手深度全景资料分析扫描记录卡】
对方角色锚定名：【 {targetPerson} 】
对手既有性格核心驱动力扫描表： 
{personaSection} 

近期交互溯源 (用于提取潜在因果、已推辞事宜，避免信息对冲漏算)：
{historySection} 

当前核心交锋第一现场 (最致命信息依据所在，请以侦探思维拆解这下面寥寥几字的压迫感与急迫度、话语里有无刺、留白):

{rawText}

请现在直接根据以上的 JSON Schema 开始吐出包含 A/B/C 三重选择对局面的结果 JSON，以协助本产品完成客户端对最终微信的代理！
```

### 这套方案对项目的高效与作用在哪里？

1.  **极度增强的 UI (商业爆点)**
    之前你的 UI 可能就是展示：“方案 A: 好的收到 / 方案 B: 目前还需要时间...”。用户虽然能用，但会觉得平庸。
    使用新的 **System Prompt 2**，它的 JSON 包含了一个核心字段：**`"tacticalGoal": "目的说明"`**。
    前端你可以在渲染每个候选气泡的时候，在其右下角或底部，放一小行灰色字体的批注！
    比如界面展示成：
    > **A. 高情商回复选项:** 
    > 王总，这边的几个核心数正在对齐收拢排雷中，今天五点整定稿给您查验。 
    > 💡 *内脑解析录：(暗指已发现存在瑕疵点因此必须卡5点以防后续问罪、平抚他需要确定性的强诉求但留缓冲以柔克刚... )*
    
    你不知道现在的国人特别是初级白领对“心智解析”、“听劝指导暗指目的”有多热衷，只要用户看一次这些大模型推演出这句话所隐含的各种“后宫博弈式的险恶意图预读解读批注”，他们一定再也放不下你的软件并且绝对疯狂发红书分享扩散给朋友展示这超高的情商。这就将外围打辅助的技术完全打入内核爽点上。

2.  **解除了字数生硬、以及脱离身份感（“你是谁的问题”）**
    你在以前用一句干巴巴“对方身份什么都没有规定”发牢骚是因为这通常靠前端做大量 UI 设置开关。让新 **Prompt 1 的 `"powerIdentity"`** 通过 AI 引擎 自动分析之前上下文并回填进变量区，相当于系统帮客户免除了“自己填写我领导如何不好怎么着”的心路操作阻断流摩擦消耗（无缝体验闭环）。
