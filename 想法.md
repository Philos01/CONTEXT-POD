\# 伴聊悬浮舱 (Context-Pod) 桌面端 AI Agent 技术白皮书 v1.0



\## 1. 产品概述 (Product Overview)

\*\*Context-Pod\*\* 是一款专为中国职场与高频社交场景设计的\*\*纯桌面端、全本地隐私保护的 AI Copilot 智能体\*\*。

它作为一个极轻量级的桌面“幽灵挂件”存在。用户在微信、企业微信或钉钉等 PC 客户端聊天时，只需一个快捷键，它就能瞬间读取当前屏幕上下文，结合用户本地建立的“联系人性格档案库（CRM）”，利用云端大模型生成极高情商的回复，并\*\*自动接管键盘将生成的回复注入到原聊天输入框\*\*中。



\*\*核心价值主张：\*\*

1\.  \*\*零摩擦交互\*\*：无需截图、无需切屏、无需复制粘贴。

2\.  \*\*绝对隐私\*\*：所有个人聊天记忆、联系人性格画像和向量化检索，全部在用户电脑本地硬盘执行（Local-First RAG）。

3\.  \*\*千人千面\*\*：不讲“正确的废话”，回复带有强烈的网感和定制化的人设护城河。



\## 2. 核心功能工作流 (Core UX Flow)

1\.  \*\*幽灵潜伏\*\*：用户在 PC 微信输入框中输入残缺语句（如“好的，但…”），但不敲击回车。

2\.  \*\*快捷唤醒\*\*：按下全局快捷键 `Alt + Space`。

3\.  \*\*无感抓取\*\*：光标旁弹出 Vue 3 渲染的毛玻璃悬浮窗，系统在 50ms 内利用系统剪贴板通道，自动读取输入框文字及前文。

4\.  \*\*本地溯源\*\*：在本地向量库中检索对方姓名（如“王总”），提取“此人脾气急躁，爱画大饼”的本地私有记忆。

5\.  \*\*推演涌现\*\*：通过 LangGraph 多智能体工作流，在悬浮窗内流式（打字机效果）呈现三种高情商回复策略（A: 顺从推进；B: 委婉甩锅；C: 幽默化解）。

6\.  \*\*一键注入\*\*：用户使用鼠标点击或数字键选择方案，悬浮窗立刻消失，系统自动将文案无缝注入并替换掉微信输入框原有的文字。

\## 3. 系统架构设计与技术栈选型 (Tech Stack)



为实现“不写复杂底层代码”的目标，我们采用“搭积木”的方式，依靠当前（2026年标准）最成熟的高集成度 NPM 包。



| 模块类别 | 选型方案 | 优势/说明 |

| :--- | :--- | :--- |

| \*\*桌面基座\*\* | \*\*Tauri v2\*\* | 比 Electron 内存占用少 80%，打包体积小（约 10MB）。提供海量开箱即用的官方 TypeScript 插件。 |

| \*\*前端框架\*\* | \*\*Vue 3 + Vite + TailwindCSS\*\* | 使用 Composition API，极为清爽。Tailwind 用于快速构建透明毛玻璃无边框悬浮窗 UI。 |

| \*\*大语言模型\*\* | \*\*DeepSeek V3 (API)\*\* 或 \*\*Kimi\*\* | 极速、低成本，国内最强上下文理解与情绪感知能力。 |

| \*\*AI 应用层\*\* | \*\*Vercel AI SDK (`@ai-sdk/vue`)\*\* | 原生支持 Vue，一个 `useCompletion` 即可接管所有复杂的流式输出 (Streaming) 与网络请求状态。 |

| \*\*工作流编排\*\* | \*\*LangGraph.js\*\* | 用于构建“理解->查库->生成”的多 Agent 思维链，结构清晰，可维护性极强。 |

| \*\*本地向量搜索\*\* | \*\*Orama\*\* (或 LanceDB Web) | 纯 JavaScript 编写的极速本地内存/浏览器端向量+全文搜索引擎，完美融入 Vue 前端。 |

| \*\*本地模型运行\*\* | \*\*Transformers.js\*\* | 直接在 Vue (WebGPU/WASM) 中运行 Embedding 模型，将文字转为向量，数据绝对不离本地电脑。 |

\## 4. 核心模块技术实现细则 (附 Vue 3 \& TS 代码逻辑)



\### 4.1 模块一：全局接管与环境感知层

\*\*目标\*\*：全局监听按键，利用系统剪贴板“偷天换日”拿数据，避免写复杂的 OCR 或图像识别代码。



\*\*实现\*\*：使用 Tauri 的官方插件 `@tauri-apps/plugin-global-shortcut` 和 `@tauri-apps/plugin-clipboard-manager`。



```typescript

// src/composables/useCapture.ts (Vue Composable)

import { register } from '@tauri-apps/plugin-global-shortcut';

import { readText, writeText } from '@tauri-apps/plugin-clipboard-manager';

import { getCurrentWindow } from '@tauri-apps/api/window';

import { invoke } from '@tauri-apps/api/core'; // 调用最微小的键盘宏辅助



export function useCapture() {

&#x20; const initShortcut = async (onTrigger: (text: string) => void) => {

&#x20;   await register('Alt+Space', async () => {

&#x20;     // 1. 唤起并聚焦 Tauri 悬浮窗口

&#x20;     const appWindow = getCurrentWindow();

&#x20;     await appWindow.show();

&#x20;     await appWindow.setFocus();



&#x20;     // 2. 模拟原生操作：调用极简的 Rust 命令执行 Ctrl+A, Ctrl+C

&#x20;     // (注：这里是唯一需要一丁点 Rust 辅助的地方，用于发送键盘指令)

&#x20;     await invoke('simulate\_copy\_action'); 



&#x20;     // 3. 从剪贴板读取抓取到的微信聊天文本

&#x20;     setTimeout(async () => {

&#x20;       const text = await readText();

&#x20;       onTrigger(text); // 将抓取到的文本传给业务流

&#x20;     }, 50); 

&#x20;   });

&#x20; };



&#x20; return { initShortcut };

}

```



\### 4.2 模块二：全本地化记忆与 RAG 层 (Transformers.js + Orama)

\*\*目标\*\*：在不需要后端服务器的情况下，让 Vue 应用直接具备人脉记忆向量检索能力。



\*\*实现\*\*：`Transformers.js` 提供本地特征提取，`Orama` 提供极致轻量的本地搜索。



```typescript

// src/services/memoryService.ts

import { pipeline } from '@xenova/transformers';

import { create, insert, search } from '@orama/orama';



// 初始化本地 Orama 向量数据库

const memoryDb = await create({

&#x20; schema: {

&#x20;   contactName: 'string',

&#x20;   personality: 'string',

&#x20;   embedding: 'vector\[384]', // 对应 all-MiniLM-L6-v2 的维度

&#x20; }

});



// 单例加载 Embedding 模型（借助浏览器的 WASM）

const extractor = await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2');



export async function addMemory(name: string, personality: string) {

&#x20; const output = await extractor(personality, { pooling: 'mean', normalize: true });

&#x20; await insert(memoryDb, {

&#x20;   contactName: name,

&#x20;   personality: personality,

&#x20;   embedding: Array.from(output.data)

&#x20; });

}



export async function retrieveMemory(queryText: string) {

&#x20; const queryVector = await extractor(queryText, { pooling: 'mean', normalize: true });

&#x20; const results = await search(memoryDb, {

&#x20;   mode: 'vector',

&#x20;   vector: { value: Array.from(queryVector.data), property: 'embedding' },

&#x20;   limit: 1

&#x20; });

&#x20; return results.hits\[0]?.document?.personality || "暂无此人记录";

}

```



\### 4.3 模块三：多智能体推演大脑 (LangGraph.js)

\*\*目标\*\*：定义一条逻辑清晰的流水线：从分析文本 -> 获取记忆 -> 融合生成。大语言模型在这里扮演核心调度。



```typescript

// src/services/agentWorkflow.ts

import { StateGraph, END } from "@langchain/langgraph";

import { ChatOpenAI } from "@langchain/openai";

import { retrieveMemory } from "./memoryService";



// 定义工作流中传递的 State

interface AgentState {

&#x20; rawText: string;

&#x20; targetPerson?: string;

&#x20; memoryData?: string;

&#x20; finalPrompt?: string;

}



const llm = new ChatOpenAI({ 

&#x20; modelName: "deepseek-chat", 

&#x20; configuration: { baseURL: "https://api.deepseek.com" } 

});



// 节点 1：抽取对话对象名字

async function extractPerson(state: AgentState) {

&#x20; // 简易正则或小模型抽取，假设抓取的文本里有 "王总: 进度？"

&#x20; const nameMatch = state.rawText.match(/(.\*?):/);

&#x20; return { targetPerson: nameMatch ? nameMatch\[1].trim() : "未知联系人" };

}



// 节点 2：查本地向量库

async function getContext(state: AgentState) {

&#x20; if (!state.targetPerson) return {};

&#x20; const mem = await retrieveMemory(state.targetPerson);

&#x20; return { memoryData: mem };

}



// 节点 3：组装最终给云端的 Prompt

async function buildPrompt(state: AgentState) {

&#x20; const prompt = `你是顶尖公关专家。你在和【${state.targetPerson}】对话。

&#x20; 他对你的印象/性格特点是：【${state.memoryData}】

&#x20; 当前聊天上下文：【${state.rawText}】

&#x20; 请直接输出三种高情商的应对短句，格式为 JSON 数组。`;

&#x20; return { finalPrompt: prompt };

}



// 拼接 LangGraph (无冗余代码，高度抽象)

const workflow = new StateGraph<AgentState>({ channels: {} })

&#x20; .addNode("extract", extractPerson)

&#x20; .addNode("memory", getContext)

&#x20; .addNode("prompt", buildPrompt)

&#x20; .addEdge("extract", "memory")

&#x20; .addEdge("memory", "prompt")

&#x20; .addEdge("prompt", END);



export const appWorkflow = workflow.compile();

```



\### 4.4 模块四：Vue 3 前端呈现与无缝回填 (Vercel AI SDK)

\*\*目标\*\*：用 Vue 3 的优雅语法展示流式打字机效果，并实现一键将结果替换至微信。



```html

const rawContext = ref('');

// Vercel AI SDK 原生支持 Vue

&#x20; api: '/api/generate', // 如果直接前端调大模型，这里替换为自己的调用逻辑

onMounted(() => {

</script>



<template>
```

\## 5. 补充说明：极少的 Rust 桥接代码

为了遵守您的要求并实现“零摩擦键盘注入”，我们需要在 Tauri 的 `src-tauri/src/main.rs` 中写入极少量的 Rust 代码来弥补前端无法控制系统键盘的短板。使用高度封装的 `enigo` 库，总代码不超过 20 行：



```rust

use enigo::{Enigo, Key, KeyboardControllable};



\#\[tauri::command]

fn simulate\_copy\_action() {

&#x20;   let mut enigo = Enigo::new();

&#x20;   // 模拟 Ctrl+A 然后 Ctrl+C

&#x20;   enigo.key\_down(Key::Control);

&#x20;   enigo.key\_click(Key::Layout('a'));

&#x20;   enigo.key\_click(Key::Layout('c'));

&#x20;   enigo.key\_up(Key::Control);

}



\#\[tauri::command]

fn simulate\_paste\_action() {

&#x20;   let mut enigo = Enigo::new();

&#x20;   // 模拟 Ctrl+V 

&#x20;   enigo.key\_down(Key::Control);

&#x20;   enigo.key\_click(Key::Layout('v'));

&#x20;   enigo.key\_up(Key::Control);

}



fn main() {

&#x20;   tauri::Builder::default()

&#x20;       .invoke\_handler(tauri::generate\_handler!\[simulate\_copy\_action, simulate\_paste\_action])

&#x20;       .run(tauri::generate\_context!())

&#x20;       .expect("error while running tauri application");

}

```



\## 6. 总结

整套《伴聊悬浮舱 (Context-Pod)》系统通过 \*\*Tauri + Vue 3\*\* 实现了极度轻量化的前端 UI，通过 \*\*Transformers.js + Orama\*\* 实现了惊艳的浏览器本地化 RAG（检索增强生成），并通过 \*\*LangGraph\*\* 将复杂的思维链固化为节点。



开发团队几乎不需要处理任何底层算法、C++ 注入代码或复杂的 WebSocket 后端服务，像搭积木一样调用现代 NPM 包，即可在几周内打磨出一款极具商业引爆潜力、保护用户隐私、且让用户爱不释手的“职场数字外挂大师”。
