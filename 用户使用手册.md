## 1. 产品概述

### 1.1 产品简介

伴聊悬浮舱是一款基于人工智能的智能聊天辅助工具，旨在帮助用户在各类即时通讯场景中生成高质量回复。产品通过深度学习技术分析对话上下文，结合用户个人风格和联系人特征，提供个性化的回复建议。

**主要特点：**

- **智能分析**：自动识别对话对象、场景和情绪
- **风格学习**：学习用户个人风格，生成符合用户习惯的回复
- **人物画像**：为每位联系人建立性格档案，实现针对性回复
- **多策略支持**：提供多种回复策略，适应不同社交场景
- **隐私保护**：所有数据本地存储，不经过第三方服务器

### 1.2 核心功能

| 功能模块 | 功能描述 | 使用场景 |
|---------|---------|---------|
| 智能回复生成 | 根据对话内容生成多种风格回复 | 收到难以回复的消息时 |
| 联系人管理 | 管理联系人信息及性格画像 | 建立和维护联系人档案 |
| 风格提取 | 从聊天记录中提取说话风格 | 分析对方或自己的沟通特点 |
| 赛博捏脸 | 通过场景题快速建立风格画像 | 快速建立个人风格模型 |
| 提示词管理 | 自定义AI回复策略 | 高级用户定制化需求 |
| 系统日志 | 记录系统运行状态 | 问题排查和性能监控 |

### 1.3 系统要求

**硬件要求：**

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 处理器 | 双核 1.6GHz | 四核 2.0GHz 及以上 |
| 内存 | 4GB RAM | 8GB RAM 及以上 |
| 存储空间 | 200MB | 500MB 及以上 |

**软件要求：**

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 10.15+、Linux |
| 网络 | 需要稳定的互联网连接 |
| 依赖服务 | 需要配置AI服务商API密钥 |

## 2. 界面介绍

### 2.1 主界面布局

伴聊悬浮舱采用简洁的悬浮窗口设计，主界面分为以下区域：

```
┌─────────────────────────────────────────────────────────┐
│  ● Context Pod                          [⚙️] [👥] [×]   │  ← 标题栏
├─────────────────────────────────────────────────────────┤
│                                                          │
│                    主内容区域                             │  ← 功能入口/回复展示
│                                                          │
├─────────────────────────────────────────────────────────┤
│  [提取风格]  [赛博捏脸]  [提示词]  [系统日志]             │  ← 功能按钮区
└─────────────────────────────────────────────────────────┘
```

**界面说明：**

- **标题栏**：显示应用名称、状态指示灯和快捷操作按钮
- **主内容区域**：根据当前状态显示不同内容（功能入口、回复建议、设置面板等）
- **功能按钮区**：快速访问各功能模块

### 2.2 功能模块入口

| 按钮 | 名称 | 功能说明 |
|------|------|----------|
| ⚙️ | 设置 | 配置API密钥、选择AI模型、设置快捷键 |
| 👥 | 联系人 | 管理聊天对象、查看编辑性格档案 |
| 提取风格 | 风格提取 | 从聊天记录中提取说话风格 |
| 赛博捏脸 | 人设配置 | 快速建立个人风格画像 |
| 提示词 | 提示词管理 | 自定义AI回复策略 |
| 系统日志 | 日志查看 | 查看系统运行记录 |

### 2.3 界面元素说明

**状态指示灯：**

| 颜色 | 状态 | 说明 |
|------|------|------|
| 绿色 | 就绪 | 系统正常，等待操作 |
| 黄色闪烁 | 工作中 | 正在生成回复或处理数据 |

**回复卡片：**

每条回复建议以卡片形式展示，包含：
- 回复内容预览
- 策略类型标签
- 点击复制按钮

## 3. 快速入门

### 3.1 首次配置

#### 步骤一：获取API密钥

1. 访问AI服务商官网注册账号
2. 进入API管理页面获取密钥

**推荐服务商：**

| 服务商 | 网址 | 特点 |
|--------|------|------|
| DeepSeek | platform.deepseek.com | 中文理解好，免费额度多 |
| OpenAI | platform.openai.com | 模型能力强 |
| 通义千问 | dashscope.aliyun.com | 国内访问稳定 |

#### 步骤二：配置应用

1. 点击界面右上角 **设置按钮**（齿轮图标）
2. 在"服务商"下拉菜单中选择对应服务商
3. 将API密钥粘贴到"API密钥"输入框
4. 点击 **保存设置**

> **注意**：API密钥将安全存储在本地，不会上传到任何服务器。

#### 步骤三：添加联系人

1. 点击 **联系人按钮**（人物图标）
2. 点击 **添加联系人**
3. 填写联系人信息：
   - **姓名**（必填）：对方的称呼
   - **身份**（选填）：如"客户"、"朋友"、"领导"
   - **性格描述**（选填）：简单描述对方性格特点
4. 点击 **添加** 完成创建

### 3.2 基础操作

#### 生成回复

1. 在聊天软件中复制对方消息（Ctrl+C）
2. 按下快捷键 **Alt+Space** 唤起悬浮舱
3. 系统自动识别剪贴板内容
4. 从联系人列表选择对应联系人
5. 点击 **确认** 开始分析
6. 查看生成的回复建议
7. 点击满意的回复，自动复制到剪贴板
8. 切换回聊天软件粘贴发送（Ctrl+V）

#### 切换回复策略

在回复结果页面，可以切换不同的回复策略：

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| A策略（顺从推进） | 积极响应，快速推进 | 需要快速推进事务时 |
| B策略（委婉甩锅） | 留有余地，争取时间 | 需要缓冲或思考时 |
| C策略（幽默化解） | 轻松氛围，化解尴尬 | 气氛紧张或需要破冰时 |

### 3.3 快速参考

| 操作 | 方法 |
|------|------|
| 唤起悬浮舱 | 按 **Alt+Space** |
| 粘贴内容 | **Ctrl+V** |
| 复制回复 | 点击回复卡片 |
| 关闭悬浮舱 | 点击 **关闭按钮** 或按 **Esc** |
| 返回主页 | 点击 **返回按钮** 或按 **Esc** |

## 4. 功能详解

### 4.1 智能回复生成

#### 功能说明

智能回复生成是伴聊悬浮舱的核心功能，通过AI分析对话内容，结合联系人画像和用户风格，生成多种回复建议。

#### 使用方法

1. **输入对话内容**
   - 方式一：复制聊天内容后按快捷键自动识别
   - 方式二：手动粘贴到输入框

2. **选择联系人**
   - 从已有联系人列表选择
   - 或手动输入联系人名称

3. **选择策略**
   - 系统默认生成三种策略的回复
   - 可点击策略按钮切换

4. **选择回复**
   - 阅读生成的回复建议
   - 点击满意的回复自动复制

#### 回复示例

**场景**：对方询问方案意见

> **对方说**："这个方案你觉得怎么样？周五之前能定吗？"

| 策略 | 生成的回复 |
|------|-----------|
| A策略 | "我觉得这个方案整体思路很清晰，有些细节我再优化一下，周四前发给你最终版？" |
| B策略 | "方案我看过了，整体方向没问题。不过有几个地方想再确认下，你方便的话晚点开个短会？" |
| C策略 | "这个方案有点意思，不过我有个更妙的想法——周四给你个惊喜怎么样？" |

#### 添加自定义回复

生成回复后，可以添加自己的回复让系统学习：

1. 点击 **添加自己的回复**
2. 输入你实际发送的回复内容
3. 点击 **保存**
4. 系统会学习你的回复风格

> **隐私说明**：你的回复仅保存在本地，用于学习你的风格，不会发送到任何服务器。

### 4.2 联系人管理

#### 功能说明

联系人管理模块用于维护联系人信息和性格画像，帮助AI生成更有针对性的回复。

#### 添加联系人

1. 点击 **联系人图标** 进入联系人管理
2. 点击 **添加联系人** 按钮
3. 填写联系人信息：
   - **姓名**（必填）：对方的称呼或名字
   - **身份**（选填）：帮助AI理解关系，如"相亲对象"、"客户"、"领导"
   - **性格描述**（选填）：描述对方性格特点
4. 点击 **添加** 完成

#### 查看联系人详情

1. 在联系人列表中找到目标联系人
2. 点击联系人卡片展开详情
3. 可查看：
   - **对话缓存**：未处理的聊天记录
   - **人物风格画像**：AI分析的性格特征

#### 编辑风格画像

1. 展开联系人详情
2. 点击 **编辑** 按钮
3. 可修改以下内容：

| 字段 | 说明 | 示例 |
|------|------|------|
| 权力身份特征 | 对方的身份定位和行为模式 | "社交能量等级：高能量人格" |
| 核心诉求 | 对方真正想要什么 | "在不同场景下需要不同的应对策略" |
| 沟通禁区 | 哪些话题或行为是雷区 | "严禁冷暴力和突然消失" |
| 情绪热度 | 对方的情绪激烈程度（0-10） | 5（中等） |
| 文本风格 | 对方的说话习惯 | "综合热度5/10，攻击性3/10" |
| 综合总结 | 对该人物的总体描述 | "太极推手 \| 亲密度:6 血缘:4 友谊:7 冲突:3" |

4. 点击 **保存修改**

#### 更新画像

1. 展开联系人详情
2. 点击 **更新画像** 按钮
3. 系统会根据对话缓存重新分析
4. 更新完成后显示处理结果

> **注意**：更新画像需要至少1条对话缓存记录。

#### 管理对话缓存

- **查看缓存**：点击展开查看详细内容
- **清除缓存**：点击清除按钮清空该联系人的对话缓存

### 4.3 风格提取

#### 功能说明

风格提取功能通过分析聊天记录，自动提取说话风格特征，包括断句习惯、用词特点、情绪色彩等。

#### 使用方法

1. 点击 **提取风格** 图标进入
2. 在聊天软件中框选对方的聊天记录（建议10-20条以上）
3. 复制聊天记录（Ctrl+C）
4. 回到悬浮舱，粘贴到左侧文本框（Ctrl+V）
5. 从下拉菜单选择对应联系人
6. 点击 **开始提取**
7. 等待AI分析完成
8. 查看提取结果

#### 提取结果说明

| 字段 | 说明 |
|------|------|
| 断句排版 | 句子长度分布、段落习惯 |
| 口癖语气 | 常用语气词、口头禅 |
| 情绪风格 | 情绪表达方式、激烈程度 |
| 词汇特征 | 用词偏好、专业术语使用 |
| 标点习惯 | 标点符号使用特点 |

#### 注意事项

- 聊天记录越多，提取结果越准确
- 建议选择单一联系人的聊天记录
- 避免混合多个对话场景的记录

### 4.4 赛博捏脸

#### 功能说明

赛博捏脸功能通过场景问答的方式，快速建立用户的个人风格画像，适合不想导入聊天记录的用户。

#### 使用方法

1. 点击 **赛博捏脸** 图标进入
2. 选择你的MBTI人格类型（可选）
3. 回答场景问题：
   - 每道题选择你最可能的回复方式
   - 或自定义输入你的回复
4. 完成所有问题后，系统自动生成风格画像
5. 查看并确认保存

#### 已有画像管理

如果已有风格画像，可以：

- **查看画像**：显示当前风格画像摘要
- **编辑画像**：手动修改画像内容
- **更新画像**：根据对话缓存重新分析
- **查看缓存**：查看自身风格的对话缓存
- **重新测试**：重新进行场景问答

### 4.5 提示词管理

#### 功能说明

提示词决定了AI生成回复的策略和风格。高级用户可以通过自定义提示词来调整AI行为。

#### 查看提示词

1. 点击 **提示词** 图标进入
2. 查看已启用的提示词模板列表
3. 点击模板查看详细内容

#### 编辑提示词

1. 在提示词列表中选择要编辑的模板
2. 修改提示词内容
3. 注意保留 `{变量名}` 格式的占位符
4. 点击 **保存**

#### 重置提示词

如需恢复默认设置：
1. 选择要重置的模板
2. 点击 **重置为默认**
3. 确认操作

> **警告**：修改提示词可能影响回复质量，建议高级用户使用。

### 4.6 系统日志

#### 功能说明

系统日志记录应用的运行状态，可用于问题排查和性能监控。

#### 查看日志

1. 点击 **系统日志** 图标进入
2. 界面显示：
   - 日志总数统计
   - 各级别日志数量（DEBUG/INFO/WARNING/ERROR）
   - 日志列表（时间、级别、模块、消息）

#### 日志级别说明

| 级别 | 说明 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | 详细执行流程 |
| INFO | 正常信息 | 操作成功记录 |
| WARNING | 警告信息 | 可恢复的异常 |
| ERROR | 错误信息 | 需要关注的错误 |

#### 筛选和搜索

- **按级别筛选**：选择特定级别查看
- **按模块筛选**：输入模块名称过滤
- **全文搜索**：输入关键词搜索

#### 导出日志

1. 点击 **导出日志** 按钮
2. 选择保存位置
3. 日志文件下载完成

> **提示**：遇到问题时，导出日志发送给技术支持可帮助快速定位原因。

## 5. 操作指南

### 5.1 日常使用流程

#### 标准使用流程

```
收到消息 → 复制内容 → 唤起悬浮舱(Alt+Space) → 选择联系人 → 
查看回复建议 → 选择满意回复 → 粘贴发送
```

#### 详细步骤

1. **接收消息**
   - 在聊天软件中收到需要回复的消息

2. **复制内容**
   - 选中对方的消息内容
   - 按 Ctrl+C 复制

3. **唤起悬浮舱**
   - 按 Alt+Space 唤起应用
   - 系统自动识别剪贴板内容

4. **选择联系人**
   - 从列表选择已有联系人
   - 或手动输入联系人名称

5. **查看回复**
   - 等待AI生成回复（通常3-10秒）
   - 阅读三种策略的回复建议

6. **选择回复**
   - 点击满意的回复卡片
   - 内容自动复制到剪贴板

7. **发送回复**
   - 切换回聊天软件
   - 按 Ctrl+V 粘贴
   - 发送消息

### 5.2 高级使用技巧

#### 让回复更像你

1. **建立个人风格画像**
   - 使用"赛博捏脸"完成场景问答
   - 或使用"风格提取"导入自己的聊天记录

2. **完善联系人档案**
   - 添加对方的基本信息和身份
   - 使用风格提取分析对方特点
   - 手动编辑和完善画像

3. **积累对话数据**
   - 使用"添加自己的回复"功能
   - 系统会自动缓存对话记录
   - 定期更新画像提高准确性

#### 处理复杂场景

**群聊场景：**

1. 单独复制目标联系人的消息
2. 明确指定联系人名称
3. 必要时先整理消息内容

**长对话场景：**

1. 复制完整的对话上下文
2. 确保包含关键信息
3. 选择合适的回复策略

**敏感话题：**

1. 选择委婉策略（B策略）
2. 注意查看回复内容是否得体
3. 必要时手动修改后发送

## 6. 常见问题

### 6.1 配置问题

#### Q：提示"API Key无效"怎么办？

**排查步骤：**

1. 检查API密钥是否正确复制，确保没有多余空格
2. 确认API密钥对应的账户状态正常
3. 检查API额度是否充足
4. 如使用代理网络，尝试关闭代理后重试
5. 重新获取新的API密钥并配置

#### Q：应该选择哪个AI服务商？

**服务商对比：**

| 服务商 | 优点 | 缺点 | 推荐场景 |
|--------|------|------|----------|
| DeepSeek | 中文理解好、免费额度多、性价比高 | 新平台 | 国内用户首选 |
| OpenAI | 模型能力强、生态完善 | 需要科学上网、价格较高 | 对质量要求高的用户 |
| 通义千问 | 国内访问稳定、阿里出品 | 中文场景优化一般 | 无法访问其他服务的用户 |

#### Q：快捷键不生效怎么办？

**解决方案：**

1. 检查快捷键是否被其他软件占用（输入法、截图工具等）
2. 尝试更换快捷键组合：
   - F9
   - Ctrl+Shift+A
   - Ctrl+Alt+Space
3. 确认应用正在运行（检查系统托盘）
4. 重启应用后重试

### 6.2 功能问题

#### Q：为什么生成的回复不相关？

**可能原因及解决方案：**

| 原因 | 解决方案 |
|------|----------|
| 未添加联系人 | 先在联系人管理中添加对方信息 |
| 未选择联系人 | 使用时从下拉菜单选择联系人 |
| 聊天记录太短 | 输入更完整的对话上下文 |
| API配置问题 | 检查API是否正常工作 |
| 网络问题 | 检查网络连接状态 |

#### Q：风格提取失败怎么办？

**解决方案：**

1. 确保聊天记录数量充足（建议10-20条以上）
2. 确保聊天记录格式正确（包含发送者和内容）
3. 检查API配置是否正确
4. 检查网络连接状态
5. 尝试减少聊天记录数量后重试

#### Q：画像更新后内容越来越多怎么办？

**说明：**

画像更新采用智能合并策略：
- 相似的特征会被合并而非重复添加
- 置信度会随观察次数缓慢增加
- 特征数量有上限控制

**建议：**
- 定期手动编辑画像，删除不需要的特征
- 避免频繁更新画像

#### Q：窗口消失了怎么办？

**解决方案：**

1. 检查系统托盘是否有应用图标
2. 双击托盘图标唤醒窗口
3. 右键托盘图标选择"显示窗口"
4. 如无响应，重启应用程序

### 6.3 账号问题

#### Q：API额度用完了怎么办？

**各服务商解决方案：**

| 服务商 | 解决方案 |
|--------|----------|
| DeepSeek | 充值或等待次月免费额度重置 |
| OpenAI | 充值账户或升级ChatGPT Plus |
| 通义千问 | 充值阿里云账户 |

#### Q：是否需要注册账号？

**说明：**

- 使用伴聊悬浮舱本身不需要注册账号
- 但需要拥有AI服务商的API账号
- API账号需要在对应服务商官网注册

## 7. 故障排除

### 7.1 问题诊断

#### 诊断流程

```
遇到问题
    ↓
查看系统日志（点击日志图标）
    ↓
根据错误级别判断
    ↓
┌─────────────────────────────────────┐
│ ERROR → 检查API配置和网络连接        │
│ WARNING → 查看警告详情               │
│ INFO → 正常信息，可忽略              │
│ DEBUG → 调试信息，开发使用           │
└─────────────────────────────────────┘
    ↓
尝试对应解决方案
    ↓
如无法解决，导出日志联系技术支持
```

### 7.2 常见错误

#### 错误代码表

| 错误信息 | 含义 | 严重程度 |
|----------|------|----------|
| API Key无效 | API密钥错误或过期 | 高 |
| 额度不足 | API调用额度用完 | 高 |
| 网络超时 | 网络连接问题 | 中 |
| 模型加载失败 | AI模型初始化问题 | 中 |
| 窗口未响应 | 程序卡死 | 中 |
| 快捷键注册失败 | 快捷键被占用 | 低 |

### 7.3 解决方案

#### 网络问题

**症状**：调用AI无响应或超时

**解决方案：**

1. 检查网络连接是否正常
2. 尝试访问其他网站确认网络状态
3. 如使用VPN，尝试关闭或切换节点
4. 确认API服务商网站可正常访问
5. 尝试切换到其他AI服务商

#### 性能问题

**症状**：生成回复很慢

**可能原因及解决方案：**

| 原因 | 解决方案 |
|------|----------|
| 网络延迟 | 检查网络质量，切换网络 |
| AI服务器繁忙 | 避开高峰期使用 |
| 输入内容过长 | 简化输入内容 |
| 本地资源不足 | 关闭其他应用释放资源 |

#### 数据问题

**症状**：联系人数据丢失

**可能原因：**

- 清理系统数据时误删
- 应用卸载重装
- 存储异常

**预防措施：**

- 定期记录重要联系人信息
- 避免清理应用相关数据
- 重要画像信息可手动备份

### 8.3 隐私安全

#### 数据存储

- **本地存储**：所有联系人数据、风格画像、对话缓存均存储在用户本地设备
- **不上传服务器**：用户数据不会上传到任何第三方服务器
- **API调用**：仅对话内容会发送到AI服务商进行处理

#### 安全措施

- API密钥本地加密存储
- 不收集用户个人信息
- 不追踪用户使用行为
- 代码开源可审计

#### 数据处理

- 对话内容仅在生成回复时发送给AI服务商
- 生成完成后不保留在服务商服务器
- 用户可随时清除本地数据
